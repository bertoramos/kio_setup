      Cómo funciona esta app

      Esta app configura beacons Kontakt.io por lote a través de la Kio Cloud API. No habla por BLE con los beacons: manda la configuración a la nube, y son los beacons (vía la app oficial o un gateway) los que la recogen.

      1. Primera vez

      1. Pestaña Ajustes → pega tu API Key (se genera en panel.kontakt.io → My Account → API Keys).
      2. Pulsa Probar conexión. Debería salir ✔ Conexión OK.
      3. Pulsa Guardar.

      2. Configurar beacons en lote

      1. Pestaña Beacons → Refrescar. Aparece tu inventario.
      2. Marca los checkboxes de los beacons que quieras cambiar.
      3. Pulsa Configurar en la barra inferior.
      4. Elige TX Power (0 = mínimo, 7 = máximo) y/o Intervalo (ms).
      5. Aplicar a seleccionados. Verás progreso por beacon.

      3. Sincronizar con los beacons físicos

      Después de aplicar cambios verás el badge ⏳ Pendiente sync. Eso quiere decir que la nube ya tiene tu nueva config, pero el beacon todavía no la ha recibido. Hay dos formas de que llegue al hardware:

      • App oficial Kio Setup (Android/iOS): ábrela cerca de los beacons. Se conecta a ellos por Bluetooth y empuja la config descargada de la nube. Es la opción más habitual sin infraestructura.
      • Gateway Kontakt (Portal Light): si tienes uno desplegado, sincroniza solo cuando el beacon esté en su rango.

      Cuando la sincronización se completa, el beacon empieza a emitir con la nueva config y en la siguiente Refrescar el badge ⏳ Pendiente sync desaparecerá.

      4. Escaneo BLE local

      La pestaña BLE usa Web Bluetooth de Chrome Android para listar dispositivos BLE cercanos (requiere HTTPS y estar cerca del beacon). Sirve para verificar que un beacon está emitiendo y ver su nombre/ID, no para configurarlo.

      Qué significa cada valor

      • TX Power: potencia de transmisión. Más bajo = menos alcance, más batería. Más alto = más alcance, menos batería. Escala 0–7 (el mapeo a dBm depende del modelo).
      • Intervalo: cada cuánto emite (en ms). Rango típico 100–10240 ms. Más alto = menos consumo, pero detección más lenta.
      • TX x y xxx ms en la línea gris: es la config actual según la nube, no lo que está emitiendo el beacon físico si hay un ⏳ Pendiente sync.

      Limitaciones honestas

      • Si no hay badge pendiente, eso solo indica que la nube no tiene cambios en cola. No confirma que el beacon físico esté emitiendo con la última config; solo lo sabrás seguro cuando un gateway/app haya sincronizado.
      • La API Key se guarda solo en este dispositivo (localStorage). Si cambias de móvil/navegador hay que volver a pegarla.
      • Si ves Error de red o CORS, revisa la Base URL en Ajustes o usa el Cloudflare Worker proxy (instrucciones en el README del repo).
