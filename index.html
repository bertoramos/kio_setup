      Cómo funciona esta app

      Esta app configura beacons Kontakt.io por lote a través de la Kio Cloud API. No habla por BLE con los beacons: manda la configuración a la nube, y son los beacons (vía la app oficial o un gateway) los que la recogen.

      1. Primera vez

      1. Pestaña Ajustes → pega tu API Key (se genera en panel.kontakt.io → My Account → API Keys).
      2. Pulsa Probar conexión. Debería salir ✔ Conexión OK.
      3. Pulsa Guardar.

      2. Configurar beacons en lote

      1. Pestaña Beacons → Refrescar. Aparece tu inventario.
      2. Marca los checkboxes de los beacons que quieras cambiar.
      3. Pulsa Configurar en la barra inferior.
      4. Elige TX Power (1 = mínimo, 7 = máximo) y/o Intervalo (ms).
      5. Aplicar a seleccionados. Verás progreso por beacon.

      3. Sincronizar con los beacons físicos (PASO OBLIGATORIO)

      ⚠️ Importante: Esta PWA solo configura la nube. Los beacons físicos no están conectados a Internet: solo emiten señales BLE. Para recibir la nueva configuración, necesitan que un dispositivo se conecte a ellos vía Bluetooth y les transfiera los cambios.

      Flujo completo de configuración

      1. Paso 1 - Configuración remota (esta PWA): Seleccionas beacons, eliges TX Power/Intervalo, y pulsas aplicar. Esto encola los cambios en la nube de Kontakt. Verás ⏳ Pendiente sync.
      2. Paso 2 - Sincronización física (obligatoria): Abres la app oficial Kio Setup Manager en Android/iOS, te acercas a cada beacon, y pulsas "Apply configuration" (o esperas a que el app lo detecte y lo haga automáticamente). La app se conecta por BLE, autentica con el protocolo propietario de Kontakt, y transfiere la configuración pendiente.
      3. Resultado: El beacon actualiza su firmware interno, empieza a emitir con los nuevos valores, y la nube marca el sync como completado.

      ¿Por qué no puedo hacerlo todo desde esta PWA?

      Los beacons no tienen WiFi/4G. El navegador web (incluso con Web Bluetooth) no puede hablar el protocolo propietario que Kontakt usa para autenticar y flashear configuración en el beacon. Ese protocolo solo está implementado en las apps nativas oficiales de Kontakt.

      Métodos de sincronización física

      • App oficial Kio Setup Manager (Android/iOS): el método estándar. Ve beacon por beacon (o usa "Apply All" si están todos al alcance), autentica, y aplica los cambios encolados en la nube. Es manual pero funciona sin infraestructura adicional.
      • Gateway Kontakt (Portal Light, Portal Beam): si tienes gateways desplegados cerca de los beacons, estos pueden hacer el sync automáticamente cuando el beacon entra en su rango. Requiere comprar y desplegar hardware adicional.

      Cuando la sincronización se completa, en la siguiente Refrescar de esta PWA el badge ⏳ Pendiente sync desaparecerá.

      4. Escaneo BLE local

      La pestaña BLE usa Web Bluetooth de Chrome Android para listar dispositivos BLE cercanos (requiere HTTPS y estar cerca del beacon). Sirve para verificar que un beacon está emitiendo y ver su nombre/ID, no para configurarlo.

      Qué significa cada valor

      • TX Power: potencia de transmisión (1–7). Más bajo = menos alcance, más batería. Más alto = más alcance, menos batería. El valor 0 puede no ser válido en algunos modelos (causa error 422). El mapeo exacto a dBm depende del modelo.
      • Intervalo: cada cuánto emite (en ms). Rango típico 100–10240 ms. Más alto = menos consumo, pero detección más lenta.
      • TX x y xxx ms en la línea gris: es la config actual según la nube, no lo que está emitiendo el beacon físico si hay un ⏳ Pendiente sync.

      Limitaciones honestas

      • Si no hay badge pendiente, eso solo indica que la nube no tiene cambios en cola. No confirma que el beacon físico esté emitiendo con la última config; solo lo sabrás seguro cuando un gateway/app haya sincronizado.
      • La API Key se guarda solo en este dispositivo (localStorage). Si cambias de móvil/navegador hay que volver a pegarla.
      • Si ves Error de red o CORS, revisa la Base URL en Ajustes o usa el Cloudflare Worker proxy (instrucciones en el README del repo).
